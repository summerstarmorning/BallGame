# 游戏开发感想与总结

## 2026-04-13: 粒子反馈与可视化强化
**今天的实现与思考：**

- **把特效接入“主工程真实路径”**：
  之前有粒子模块，但主游戏循环里没有直接使用，玩家运行后感知不到效果。这次改成直接在 `Game` 中维护粒子队列，和小球边界碰撞联动，保证“改了就能看到”。

- **反射不是只做一次，而是持续模拟**：
  粒子不只是生成后直线飞散，而是会在屏幕四周继续反弹并逐步衰减，这样视觉上更有“能量回弹”的感觉。

- **性能与效果平衡**：
  增加了粒子数量上限、寿命回收、透明度渐隐，尽量在观感提升和稳定帧率之间保持平衡。

## 2026-04-02: 第一次迭代与手感优化
**今天遇到的挑战与反思：**

- **Git 同步与网络问题**：
  刚开始开发遇到了一个很难受的 `Connection was reset` HTTPS 连接错误卡住了代码版本管理。通过使用详细诊断（`GIT_CURL_VERBOSE`）确认了握手过程的问题，最终改用本地生成的 SSH 密钥直接绕过了拦截。这反映了在遇到诡异的网络拦截时，SSH 是一个很好的备用或首选方案。

- **C++ 中文乱码与库命名空间冲突**：
  在尝试修复 Windows 控制台中文乱码时，一个下意识的想法是引入 `<windows.h>`，结果直接暴露了 `raylib.h` 原本和 Windows API 的函数重名冲突（比如 `CloseWindow`）。最后使用 `std::system("chcp 65001 > nul")` 这个折中又直接的做法完美化解了冲突。

- **物理碰撞手感的深度调优**：
  学习了如何提升“打砖块”游戏的手感：绝不是单纯地用内置交叉检测去“碰到就反弹”。如果圆和矩形重叠导致一帧里多次取反，就会卡住。引入了“只在向下掉时反弹速度”，并且给球加上了“打击点偏移”和“挡板移动速度的影响”，允许玩家主动控制球的弹射角度。这小小的一步，极大提升了游戏性。

- **游戏机制形成闭环**：
  把单一的弹力球加上了真正的游戏目标和失败惩罚机制。一旦加上 Score、Lives 和 GameOver，这就变成了一个真正的“游戏雏形”。
