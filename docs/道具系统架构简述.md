# 道具系统架构简述（扩展版）

道具系统采用“实体层 + 行为层 + 管理层 + 配置层”的分层结构，目标是保证功能可扩展、参数可配置、生命周期可控。

`PowerUp` 负责道具实体本身：生成、下落、碰撞拾取、离屏清理等运行时状态，不直接包含具体效果逻辑。  
`PowerUpEffect` 是行为抽象基类，定义 `apply / update / expire` 生命周期；具体效果（如加长挡板、多球、减速）由各子类独立实现。  
`PowerUpFactory` 负责按类型创建效果对象，避免主循环中堆积 if-else，新增道具时只需注册新类型。  
`PowerUpSystem` 作为协调器，统一管理掉落道具与激活效果，处理持续时间、过期回收、跨关保留与关卡切换重挂载。

数据层由 `powerups.json` 驱动，核心参数包括：掉落概率、持续时间、强度倍率、是否永久生效。通过配置可快速调平衡，不需要重新编译。

在生命周期策略上，系统支持：
- 道具实体自动清理（拾取后或掉出屏幕）
- 临时效果倒计时到期自动恢复
- 永久效果持续保留
- 跨关时按策略决定是否重应用（例如多球可配置为不重复触发）

该架构可在不破坏既有逻辑的前提下持续扩展新道具类型，适合后续叠加 UI 提示、调试信息与关卡差异化玩法。
